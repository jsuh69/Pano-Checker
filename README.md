# Pano-Checker

M5StickS3 (ESP32-S3 기반) 모터 RPM 측정 및 주파수 분석기

## v0.1 주요 업데이트 내역 (최근 수정 사항)

### 1. 배터리 충전 안정성 개선 및 최적화
* **충전 사이클 정상화**: 기기가 켜진 상태에서 충전이 제대로 되지 않던 문제 해결. (PMIC의 충전 상태 머신이 매 루프마다 초기화되는 현상을 10초 타이머로 제어)
* **전압 측정 최적화**: 충전 시 배터리 전압이 비정상적으로 높게 측정되는 현상(IR Drop)을 우회하기 위해 정밀 측정 주기를 2분으로 조정하고, 전압 안정화 대기 시간을 3초로 단축시켜 배터리 효율 극대화.

### 2. 절전 기능 극대화 (스마트 슬립 모드)
* **오디오 및 연산 중지**: 설정된 시간 동안 입력이 없어 화면이 꺼질 경우, 무거운 FFT 주파수 분석 연산을 전면 중지하고 CPU를 대기(Idle) 모드로 전환하여 배터리 소모를 획기적으로 줄임.
* **마이크 전원 제어**: 절전 모드 진입 시 마이크 하드웨어를 아예 끄고(`M5.Mic.end()`), 기기를 깨울 때 다시 켜는(`M5.Mic.begin()`) 하드웨어 단위의 전력 제어 로직 적용.
* **스마트 버튼 UX**: 화면이 꺼져있는 상태에서 버튼(BtnA, BtnB)을 누르면 설정값이 변경되지 않고 기기를 깨우는 역할만 하도록 사용자 경험(UX) 개선.

### 3. 코드 구조 대대적 리팩토링 및 모듈화
* **상수(매직 넘버) 분리**: UI 색상, 배터리 관련 수치, 오디오 설정 등을 각각의 네임스페이스(`BatConfig`, `UI`, `Audio`, `PowerSave`)로 묶어서 가독성과 관리 편의성 증대.
* **배터리 매니저 클래스화**: 흩어져 있던 배터리 측정 변수 및 복잡한 타이머 로직들을 `BatteryManager` 클래스로 깔끔하게 캡슐화.
* **메모리 최적화**: RPM 숫자를 콤마(,)와 함께 포맷팅할 때, 메모리 파편화를 유발하던 잦은 `String` 객체 조작 대신 C-style 문자열(`char` 배열)을 활용해 최적화 달성.
